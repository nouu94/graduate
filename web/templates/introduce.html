Introduce

Purpose

Current real estate applications provide real estate information based on user interface for a large number of users. This method provides users with a visible and easy-to-manipulate UI so that customers can easily understand the information they want. However, the disadvantage of an application is that it cannot meet the needs of information that each user needs with a unified information item, and that each application has different interface controls, so it needs to learn how to operate it. The project was planned to break down the two disadvantages of these real estate applications.

Service

A simulation of the situation was conducted based on the assumption that a customer who wants to know the real estate specific information makes a request. Unlike the ability to provide interfaces to a large number of unspecified customers, such as existing real estate applications, a subscription feature was created on a webpage, allowing any user to enter an e-mail and click the subscribe button to store it in the database. Python flask was used to insert subscription function into web page and based on MVC architecture to facilitate maintenance.
